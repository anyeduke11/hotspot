## 1. 概述

### 1.1 目标

在现有 SecNews 热点地图基础上，新增「资讯 → 知识 → 掌握 → 内容创作」全流程看板。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **编译优于检索** | 知识条目一次性编译为结构化 Markdown，不走 RAG |
| **Markdown 为真相源** | Agent 读写 .md 文件，后端同步到 SQLite |
| **Agent 不嵌入后端** | 后端做数据同步/存储/API，Agent 做 LLM 调用/编译/学习 |
| **LLM-Wiki 联邦** | Hotspot wiki + 本地 wiki 交叉引用，统一展示 |
| **现有系统不动** | 资讯看板、采集器、质量门禁全部保留 |

### 1.3 决策汇总

| # | 决策点 | 选择 |
|---|--------|------|
| 1 | 编译时机 | Agent 扫描 + 每日定时 + 每周至少 1 次 + 前端手动 |
| 2 | Wiki 层级 | 完整 L1-L4 金字塔 |
| 3 | 图谱范围 | 双 wiki 合并 + 领域/主题/时间筛选 |
| 4 | 掌握度回流 | Agent 写文件，后端同步 |
| 5 | 创作日历 | 全链路：选题→草稿→发布→数据回收 |
| 6 | 健康度审计 | 完整审计，支持单/全量 wiki |
| 7 | 批量导入 | 先入库后编译，去重 + 代理验证 |
| 8 | 任务状态 | 轮询 + 手动刷新 |
| 9 | 推进方式 | 更细粒度拆分，框架可扩展 |
| 10 | Skill 集成 | 集成已有 skill，预留扩展 |
| 11 | 数据存储 | Markdown + SQLite 双存储 |
| 12 | 内容创作 | 嵌入模式，同一页面三栏 |
| 13 | 页面布局 | 知识管理 Tab 内嵌创作入口 |

## 3. 数据模型

### 3.1 知识条目 (knowledge/items/{hash}.md)

```yaml
---
id: "a1b2c3"
title: "RSA 2026 零信任架构新趋势"
source: "cubox"                    # cubox | bookmark | secnews | secnews_archive
source_url: "https://..."
ingested_at: "2026-07-14T10:00:00Z"
compiled: false                     # 是否已编译

# 分类 (4 维)
domain: "security"                  # security | ai | finance | product | engineering | ...
topic: "zero-trust"                 # 二级主题
type: "news"                        # news | analysis | paper | tutorial | tool | opinion
difficulty: "intermediate"          # beginner | intermediate | advanced | expert

# 标签 (多维)
tags:
  - rsa-conference
  - network-security
  - architecture

# 提取的概念
concepts:
  - zero-trust-architecture
  - micro-segmentation

# 学习状态
mastery: 0
last_reviewed: null
review_count: 0

# 关联
related_items:
  - "d4e5f6"
---

# RSA 2026 零信任架构新趋势

...正文...
```

### 3.9 SQLite 新增表

```sql
-- 知识条目索引 (镜像 knowledge/items/*.md)
CREATE TABLE IF NOT EXISTS knowledge_items (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    source TEXT NOT NULL,
    source_url TEXT,
    domain TEXT,
    topic TEXT,
    type TEXT,
    difficulty TEXT,
    tags TEXT,           -- JSON array
    concepts TEXT,        -- JSON array
    mastery INTEGER DEFAULT 0,
    compiled INTEGER DEFAULT 0,
    ingested_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- 概念索引 (镜像 knowledge/concepts/*.md)
CREATE TABLE IF NOT EXISTS knowledge_concepts (
    slug TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    domain TEXT,
    source_items TEXT,    -- JSON array
    local_wiki_ref TEXT,
    updated_at TEXT NOT NULL
);

-- 知识图谱缓存
CREATE TABLE IF NOT EXISTS knowledge_graph (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    graph_data TEXT NOT NULL,  -- JSON
    updated_at TEXT NOT NULL
);

-- 学习计划
CREATE TABLE IF NOT EXISTS knowledge_plans (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    week TEXT NOT NULL,
    status TEXT DEFAULT 'active',
    plan_data TEXT NOT NULL,  -- JSON
    created_at TEXT NOT NULL
);

-- 学习进度
CREATE TABLE IF NOT EXISTS knowledge_progress (
    concept_slug TEXT PRIMARY KEY,
    mastery INTEGER DEFAULT 0,
    last_tested TEXT,
    test_count INTEGER DEFAULT 0,
    updated_at TEXT NOT NULL
);

-- 创作日历
CREATE TABLE IF NOT EXISTS content_calendar (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,
    topic TEXT NOT NULL,
    type TEXT,
    status TEXT DEFAULT 'planned',
    source_items TEXT,    -- JSON array
    draft_path TEXT,
    platform TEXT,
    published_url TEXT,
    stats TEXT,           -- JSON
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- 创作草稿
CREATE TABLE IF NOT EXISTS content_drafts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path TEXT NOT NULL UNIQUE,
    title TEXT NOT NULL,
    status TEXT DEFAULT 'draft',
    calendar_id INTEGER REFERENCES content_calendar(id),
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- 任务队列
CREATE TABLE IF NOT EXISTS knowledge_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_type TEXT NOT NULL,
    status TEXT DEFAULT 'pending',
    params TEXT,          -- JSON
    result_path TEXT,
    error_message TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- Skill 配置 (LLM 模型绑定)
CREATE TABLE IF NOT EXISTS knowledge_skill_config (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_name TEXT NOT NULL UNIQUE,
    secret_id INTEGER REFERENCES secrets(id),
    model_override TEXT,
    prompt_template TEXT,
    enabled INTEGER DEFAULT 1,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

## 6. 数据清理层

### 6.1 DataCleaningService

```
输入: 三源数据 (Cuboz cards / 书签 JSON / secnews archive)
  │
  ├── Step 1: URL 指纹去重
  │   算法: SimHash (64-bit)
  │   规则: 同 URL → 合并 sources; 相似 URL (>90%) → 标记候选
  │
  ├── Step 2: 元数据合并
  │   优先级: Cubox (有全文) > 书签 (有文件夹) > secnews (有摘要)
  │   合并规则: 高优源覆盖低优源的对应字段
  │
  ├── Step 3: 来源追溯
  │   存储: sources 字段 (JSON array)
  │   示例: ["cubox", "secnews_archive"]
  │
  ├── Step 4: 代理验证
  │   书签 URL: 通过 127.0.0.1:7897 代理验证有效性
  │   无效 URL: 标记为 dead_link，不阻断导入
  │
  └── 输出: knowledge/items/{hash}.md
```

### 6.2 三源关联

一个 URL 可能同时存在于多个源：

```
URL: https://example.com/zero-trust-2026
  ├── Cubox: 有全文 + 标注 + AI insight
  ├── 书签: 在 "安全/零信任" 文件夹
  └── secnews: 有资讯摘要 + 来源评分

合并结果:
  knowledge/items/{hash}.md
  sources: ["cubox", "bookmark", "secnews_archive"]
  content: Cubox 全文 (最高优先级)
  folder: 书签文件夹 (补充元数据)
  summary: secnews 摘要 (补充摘要)
```

## 7. 编译飞轮

### 7.1 触发时机

| 触发方式 | 说明 |
|----------|------|
| Agent 按需扫描 | Agent 启动时扫描 `knowledge/items/` 中 `compiled: false` 的条目 |
| 每日定时 | 北京时间 02:00 批量编译当天增量 |
| 每周至少 1 次 | 每周日 03:00 全量增量编译 |
| 前端手动 | 用户点击「编译知识库」按钮 |

### 7.2 编译流程

```
Agent 扫描 knowledge/items/ 中 compiled=false 的条目
  │
  ├── 批量: 10 条/批
  │
  ├── Step 1: 分类 + 打标
  │   LLM: 读取标题 + 正文 → 输出 domain/topic/type/difficulty + tags
  │   写入: 条目 frontmatter
  │
  ├── Step 2: 概念提取
  │   LLM: 读取全文 → 提取关键概念
  │   写入: concepts/{slug}.md (新建或更新)
  │
  ├── Step 3: 概念关联
  │   LLM: 新概念 ↔ 已有概念 → 建立关联边
  │   写入: 条目 frontmatter.concepts + concepts/graph.json
  │
  ├── Step 4: 索引更新
  │   写入: _MAP.md (全量索引)
  │
  ├── Step 5: 角色画像更新
  │   条件: 每周至少 1 次
  │   写入: SOUL.md
  │
  └── 标记: compiled=true
```

### 7.3 增量编译

```
重复编译检测:
  - 条目 compiled=true → 跳过
  - 条目 .md 文件 mtime > compiled_at → 重新编译
  - 概念 .md 已存在 → 更新 (追加关联条目，不覆盖)
```

### 8.1 AGENTS.md 协议

```markdown
# 知识管理

## 联邦 LLM-Wiki

### 1. Hotspot LLM-Wiki (资讯→知识)
  路径: knowledge/
  数据源: Cubox / 浏览器书签 / secnews 归档
  维护者: Agent (knowledge-master skill)

### 2. 本地 LLM-Wiki (知识沉淀)
  路径: ~/knowledge-base/
  维护者: Agent (knowledge-base-manager skill)

## 联邦规则

- 概念对齐: 两个 wiki 共享同一套概念体系
- 交叉引用: [[wiki:local:...]] / [[wiki:hotspot:...]]
- 联合搜索: Agent 搜索时同时检索两个 wiki
- 知识迁移: mastery > 80% 的条目 → 归档到本地 wiki L2
```

### 8.2 交叉引用语法

```
Hotspot → 本地: [[wiki:local:concepts/zero-trust]]
本地 → Hotspot: [[wiki:hotspot:items/a1b2c3]]
```

双 wiki 知识图谱合并，颜色区分：
- 实心节点 = Hotspot wiki
- 空心节点 = 本地 wiki
- 虚线边 = 联邦交叉引用
